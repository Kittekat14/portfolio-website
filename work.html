Node.js (Movie API)

Working with Node.js and other server-side frameworks like for example ExpressJS, I built a REST API with URL endpoints, that connect to data from a MongoDB Database, which I set up myself as well. The API endpoints are tested in Postman. The API includes user authentication and authorization as well as the business logic layer using Mongoose. I incorporated server-side data validation using the express-validator. The API is fully documented using JSDoc and deployed on Heroku.

See Project on GitHub

React (Movie App)

I used React and React Bootstrap to build the single-page-application "ActorInspector" based on my existing server-side code (my RESTful Movie-API and MongoDB database). In the login and register forms, the user gets immediate feedback about his filled in credentials (username and password) through client-side validation. This Movie App includes routing between different class and functional components and is responsive.

See Project on GitHub

Serverless, Progressive Web Application (PWA) "Meet"

Inside a test-driven development environment, I created serverless functions to get or refresh an access-token to the Google API. I then build a PWA that reveives data about events from the Google Calendar API. I added a scatterchart to visualize the data.

See Project on GitHub See deployed version See Video-Demo on YouTube

React Native Mobile Chat App

I built a Chat App using React Native, a React framework, that can be bundled into Native code. Additionally, I used Expo.io as a development environment to see the views on mobile devices. The Chat App messages are being stored in the database "Firestore", a service of Firebase. The sent images are stored in Firebase's Cloud "Storage". It is optimized for both iOS and Android devices.

See Project on GitHub
